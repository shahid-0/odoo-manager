# Contributing to Odoo Manager

First off, thank you for considering contributing to Odoo Manager! It's people like you that make Odoo Manager such a great tool.

## How Can I Contribute?

### Reporting Bugs
Before creating bug reports, please check the [existing issues](https://github.com/your-username/odoo-manager/issues) as you might find out that you don't need to create one. When you are creating a bug report, please include as many details as possible by following the [bug report template](.github/ISSUE_TEMPLATE/bug_report.yml).

### Suggesting Enhancements
Enhancement suggestions are tracked as [GitHub issues](https://github.com/your-username/odoo-manager/issues). Create an issue and provide the following information:
* Use a clear and descriptive title.
* Provide a step-by-step description of the suggested enhancement.
* Describe the current behavior and explain which behavior you expected to see instead and why.

### Your First Code Contribution
1. Fork the repo.
2. Clone your fork.
3. Create a branch for your feature or fix.
4. Make your changes.
5. Push to your fork and submit a pull request.

## Style Guidelines
* We use TypeScript for the backend and React for the frontend.
* Follow the existing code style and naming conventions.
* Ensure your code is well-commented.

## Questions?
If you have questions, please feel free to open a Discussion on GitHub.
